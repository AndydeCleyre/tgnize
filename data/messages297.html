But after the Non-lexical lifetimes were introduced, The compiler got a bit smarter. It checks the code and finds out, r1,r2 and r3 have the same scope but r1 and r2 are not used after r3 has been created, So, The program is not violating the law. i.e. holding a mutable reference while it has a immutable reference.
